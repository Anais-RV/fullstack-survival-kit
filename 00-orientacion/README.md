# Orientación

Antes de escribir código, necesitas contexto.

Este módulo te prepara para el viaje: entenderás qué es el desarrollo fullstack, cómo navegar este repositorio, qué herramientas necesitas y cómo configurarlas.

**No hay programación todavía.** Solo preparación. Pero es tiempo bien invertido.

---

## ¿Por qué empezar por aquí?

Porque empezar a programar sin contexto es como intentar armar muebles de IKEA sin leer las instrucciones: técnicamente posible, pero terminarás frustrado con piezas sobrantes.

Este módulo responde:
- ¿Qué voy a aprender exactamente?
- ¿Cómo está organizado el material?
- ¿Qué herramientas necesito?
- ¿Cómo uso cada herramienta?

**Tiempo estimado:** 1-2 días (incluyendo instalaciones)

---

### [02-que-es-fullstack.md](./02-que-es-fullstack.md)
**Descripción:** Qué significa "fullstack" en la práctica. Sin marketing, sin buzzwords. Con analogías cotidianas que tienen sentido.

**Duración:** 20 minutos de lectura

**Aprenderás:**
- Frontend, backend y base de datos explicados con la analogía del restaurante
- Qué hace un desarrollador fullstack realmente
- Por qué aprender fullstack (y por qué no es obligatorio)
- El stack tecnológico que usaremos: Python + React

---

### [03-mapa-del-recorrido.md](./03-mapa-del-recorrido.md)
**Descripción:** Vista completa de los 10 módulos del repositorio. Qué aprenderás en cada uno, cómo se conectan, cuánto tiempo llevará.

**Duración:** 25 minutos de lectura

**Aprenderás:**
- Progresión completa del repositorio
- Hitos del aprendizaje
- Puntos críticos donde no puedes avanzar sin entender lo anterior
- Expectativas realistas de tiempo

**Lee esto para entender el viaje completo antes de empezar.**

---

### [04-configuracion-entorno.md](./04-configuracion-entorno.md)
**Descripción:** Instalación paso a paso de todas las herramientas: Python, Node.js, VSCode, Git. Instrucciones específicas para Windows, macOS y Linux.

**Duración:** 30-60 minutos (dependiendo de tu conexión)

**Instalarás:**
- Python 3.11+
- Node.js (para herramientas de frontend)
- Visual Studio Code (editor)
- Git (control de versiones)
- Extensiones esenciales de VSCode
- Configuración recomendada

**Incluye sección de troubleshooting** para problemas comunes.

**Este es el único módulo donde harás cosas técnicas.** El resto es solo lectura.

---

### [05-herramientas-esenciales.md](./05-herramientas-esenciales.md)
**Descripción:** Para qué sirve cada herramienta, cómo usarla, comandos esenciales. VSCode, Terminal, Git, Node.js, Python y navegador explicados en profundidad.

**Duración:** 30 minutos de lectura

**Aprenderás:**
- Atajos de VSCode que te ahorrarán horas
- Comandos de terminal que usarás todos los días
- Flujo básico de Git
- Cómo trabajan todas las herramientas juntas
- Errores comunes de principiantes (y cómo evitarlos)

**Lee esto después de instalar todo.** Te ayudará a entender qué acabas de instalar y para qué sirve.

---

### [06-git-github-esencial.md](./06-git-github-esencial.md) ⭐ **IMPRESCINDIBLE**
**Descripción:** Control de versiones con Git y GitHub. La herramienta más importante después de tu editor de código. Sin esto, no puedes trabajar profesionalmente.

**Duración:** 45-60 minutos de lectura + práctica

**Aprenderás:**
- Qué es Git y por qué es absolutamente obligatorio
- Comandos esenciales: `init`, `add`, `commit`, `push`, `pull`
- Crear tu primer repositorio en GitHub
- Flujo de trabajo con ramas (branches)
- Pull Requests para trabajo en equipo
- Crear tu perfil profesional de GitHub (tu CV técnico)
- Convenciones de commits profesionales
- Resolver conflictos de merge
- `.gitignore` y qué archivos NO subir

**⚠️ CRÍTICO:** Git/GitHub no es opcional. Es la herramienta #1 de la industria. TODO tu código debe estar en Git desde el día 1.

**Tu perfil de GitHub ES tu portfolio.** Los reclutadores lo revisan antes que tu CV.

## Notas importantes

### Este módulo no tiene ejercicios
Es solo contexto y preparación. Los ejercicios empiezan en el módulo 01-fundamentos.

### No te saltes la configuración
Puedes tener prisa por programar, pero si tu entorno no está bien configurado, perderás más tiempo después troubleshooteando problemas tontos.

### Vuelve aquí cuando necesites
Estos documentos están diseñados para consultarlos. Si en el módulo 05 olvidas un comando de Git, vuelve a [05-herramientas-esenciales.md](./05-herramientas-esenciales.md).
