# Mapa del recorrido

Ya sabes qué es fullstack y cómo navegar este repositorio. Ahora veamos **el camino completo** que recorrerás.

No es un sprint. Es una ruta de senderismo con paradas claras.

---

## Vista general: los 10 módulos

```
00-orientacion          ← Estás aquí
    ↓
01-fundamentos          ← Base de programación
    ↓
02-frontend             ← Interfaces con React
    ↓
03-backend              ← Servidores con Python
    ↓
04-bases-de-datos       ← Guardar datos con SQL
    ↓
05-integracion-fullstack ← Conectar todo
    ↓
06-testing              ← Asegurar calidad
    ↓
07-arquitectura         ← Organizar proyectos grandes
    ↓
08-despliegue           ← Publicar en internet
    ↓
09-proyecto-integrador  ← Aplicación completa
```

**Cada módulo se construye sobre los anteriores.** No es un buffet libre donde eliges al azar. Es una escalera: si saltas escalones, te caes.

---

## Los módulos en detalle

### 00 - Orientación (1-2 días)

**Qué aprenderás:**
- Cómo está organizado el repositorio
- Qué significa fullstack en la práctica
- Qué herramientas necesitas instalar
- Cómo configurar tu entorno de desarrollo

**Objetivo:**
Empezar con las expectativas claras y las herramientas listas.

**No hay código todavía.** Solo contexto y preparación.

---

### 01 - Fundamentos (3-4 semanas)

**Qué aprenderás:**
- Variables, funciones, condicionales, bucles
- Estructuras de datos (arrays, objetos)
- Asincronía (promesas, async/await)
- Fundamentos web (HTTP, APIs, JSON)

**Objetivo:**
Construir la base técnica que necesitas para todo lo demás.

**Este módulo es crítico.** Si lo saltas, el resto será una lucha constante. Si lo entiendes bien, todo lo demás fluye.

**Analogía:** Es como aprender el alfabeto antes de leer libros. Puedes intentar leer sin saberlo, pero será frustrante.

---

### 02 - Frontend (3-5 semanas)

**Qué aprenderás:**
- HTML semántico y accesible
- CSS (layouts, responsive, flexbox, grid)
- JavaScript en el navegador (DOM, eventos)
- React (componentes, estado, props)
- Formularios y validación
- Peticiones HTTP desde el frontend

**Objetivo:**
Crear interfaces completas que se vean bien y funcionen en cualquier dispositivo.

**Al final de este módulo:**
Puedes hacer una aplicación frontend que consuma datos de una API externa (aunque no la hayas creado tú todavía).

**Analogía:** Aprendes a diseñar la tienda: dónde poner las cosas, cómo organizarlas, qué botones tiene cada estantería.

---

### 03 - Backend (3-5 semanas)

**Qué aprenderás:**
- Qué es un servidor y cómo funciona
- Crear APIs REST con Python
- Django (framework completo) o FastAPI (framework moderno)
- Validación de datos
- Autenticación y autorización
- Manejo de errores
- Programación Orientada a Objetos aplicada

**Objetivo:**
Construir servidores que gestionen lógica, procesen datos y respondan peticiones de forma segura.

**Al final de este módulo:**
Puedes crear un backend completo con autenticación, CRUD de recursos y lógica de negocio.

**Analogía:** Aprendes a gestionar la cocina: recibir pedidos, validar ingredientes, preparar platos, gestionar el inventario.

---

### 04 - Bases de datos (2-3 semanas)

**Qué aprenderás:**
- Qué es una base de datos y por qué la necesitas
- SQL (el lenguaje universal de datos)
- PostgreSQL (la base de datos que usaremos)
- Diseño de tablas y relaciones
- Consultas (SELECT, INSERT, UPDATE, DELETE)
- Migraciones y modelos con Django/FastAPI

**Objetivo:**
Guardar y recuperar datos de forma eficiente y organizada.

**Al final de este módulo:**
Puedes diseñar esquemas de datos y hacer consultas complejas.

**Analogía:** Aprendes a organizar el almacén: qué estantería es para qué, cómo etiquetar las cajas, cómo encontrar cosas rápido.

---

### 05 - Integración Fullstack (2-3 semanas)

**Qué aprenderás:**
- Conectar frontend React con backend Python
- CORS (permitir que se comuniquen)
- Autenticación end-to-end (login desde frontend)
- Estado global en frontend (Context API o Zustand)
- Gestión de tokens y sesiones
- Desplegar frontend y backend juntos

**Objetivo:**
Hacer que todas las piezas trabajen juntas como un sistema cohesionado.

**Al final de este módulo:**
Tienes una aplicación fullstack completa funcionando: frontend + backend + base de datos.

**Analogía:** Conectas la sala del restaurante con la cocina y el almacén. El camarero toma pedidos, la cocina prepara, el almacén provee.

---

### 06 - Testing (1-2 semanas)

**Qué aprenderás:**
- Por qué testear (y cuándo)
- Tests unitarios (probar funciones aisladas)
- Tests de integración (probar flujos completos)
- Jest para frontend, Pytest para backend
- Cobertura de tests
- TDD básico (Test-Driven Development)

**Objetivo:**
Asegurar que tu código funciona y seguirá funcionando cuando lo cambies.

**Al final de este módulo:**
Puedes escribir tests para tus aplicaciones y entender qué merece la pena testear.

**Analogía:** Poner cámaras y sistemas de control de calidad en el restaurante para asegurar que todo funciona como debe.

---

### 07 - Arquitectura (1-2 semanas)

**Qué aprenderás:**
- Cómo organizar código en proyectos grandes
- Patrones de diseño aplicados (MVC, Repository, Service Layer)
- Separación de responsabilidades
- Clean Code en la práctica
- Refactorización sin romper nada

**Objetivo:**
Escribir código que otra persona (o tú en 6 meses) pueda entender y mantener.

**Al final de este módulo:**
Entiendes cómo estructurar proyectos profesionales, no solo scripts que funcionan.

**Analogía:** Diseñar los procesos del restaurante para que cualquier chef nuevo pueda entrar y saber dónde está todo.

---

### 08 - Despliegue (1-2 semanas)

**Qué aprenderás:**
- Qué significa "desplegar"
- Servicios de hosting (Vercel, Railway, Render)
- Variables de entorno
- Configuración de producción vs desarrollo
- Dominios y DNS básico
- CI/CD básico (automatizar el despliegue)

**Objetivo:**
Publicar tu aplicación en internet para que cualquiera pueda usarla.

**Al final de este módulo:**
Tienes tu aplicación online con una URL pública.

**Analogía:** Abrir el restaurante al público en una ubicación real, no solo en tu cocina de pruebas.

---

### 09 - Proyecto Integrador (3-4 semanas)

**Qué aprenderás:**
- Aplicar TODO lo anterior en un proyecto real
- Tomar decisiones de diseño
- Enfrentar problemas sin guía paso a paso
- Iterar sobre tu propio código

**Estructura:**
El proyecto se construye en **7 iteraciones** incrementales:
1. Configuración inicial
2. Backend básico
3. Frontend básico
4. Integración
5. Autenticación
6. Funcionalidades completas
7. Mejoras y despliegue

**Objetivo:**
Consolidar todo el aprendizaje en algo tangible que puedas mostrar.

**Al final de este módulo:**
Tienes una aplicación fullstack completa, testeada, desplegada y funcional. Algo que puedes poner en tu portfolio.

**Analogía:** Gestionar todo el restaurante de principio a fin, desde el diseño hasta servir los primeros clientes reales.

## ¿Cómo saber si estoy listo para el siguiente módulo?

### Señales de que puedes avanzar:
✅ Haces los ejercicios sin mirar constantemente el material  
✅ Puedes explicar los conceptos con tus propias palabras  
✅ Cuando algo falla, sabes dónde buscar  
✅ Modificas ejemplos y entiendes por qué se rompen (o por qué funcionan)  

### Señales de que necesitas más tiempo:
⚠️ Los ejercicios te resultan completamente confusos  
⚠️ Copias código sin entender qué hace cada línea  
⚠️ Los mensajes de error te paralizan  
⚠️ No puedes explicar qué hace el código que escribiste ayer  

**No pasa nada si necesitas más tiempo.** Mejor consolidar bien que avanzar con grietas en la base.

## ¿Puedo hacer varios módulos a la vez?

**No es recomendable.**

Cada módulo introduce conceptos nuevos que necesitan asentarse. Si intentas aprender React mientras estás lidiando con bucles en fundamentos, tu cerebro se satura.

**Mejor estrategia:**
1. Termina un módulo completo
2. Dale unos días para asentar
3. Empieza el siguiente

**Excepción:**
Si ya dominas un área (por ejemplo, vienes con experiencia en frontend), puedes saltarte ese módulo o ir más rápido. Pero no hagas dos módulos nuevos simultáneamente.

**Piensa en aprender a conducir:** No intentas aprender a aparcar y a hacer cambios de marcha al mismo tiempo en tu primera clase. Una cosa, luego la otra.

## Los puntos de no retorno

Hay momentos clave donde **no puedes avanzar sin haber entendido lo anterior**:

### Punto crítico 1: Asincronía (en Fundamentos)
Si no entiendes promesas y async/await, el resto será un infierno. Frontend y backend dependen completamente de esto.

### Punto crítico 2: React Básico (en Frontend)
Si no entiendes componentes, props y estado, el resto de React no tendrá sentido.

### Punto crítico 3: APIs REST (en Backend)
Si no entiendes cómo funciona una API, la integración fullstack será imposible.

### Punto crítico 4: SQL Básico (en Bases de datos)
Si no puedes hacer consultas básicas, no podrás trabajar con datos reales.

**Si te atascas en estos puntos:** Para, repasa, practica más. No sigas adelante con dudas. Pregunta, busca recursos adicionales, vuelve a leer. Pero no los saltes.

## Módulos interdependientes

Algunos módulos necesitan otros de forma absoluta:

```
Fundamentos
    ├─→ Frontend (requiere JS, asincronía, DOM)
    ├─→ Backend (requiere JS/Python, asincronía, HTTP)
    └─→ Bases de datos (requiere lógica de programación)

Frontend + Backend + BD
    └─→ Integración (requiere los tres anteriores)

Integración
    ├─→ Testing (necesitas algo que testear)
    ├─→ Arquitectura (necesitas código que organizar)
    └─→ Despliegue (necesitas algo que desplegar)

Todo lo anterior
    └─→ Proyecto integrador
```

**No puedes hacer el proyecto integrador si no has pasado por los módulos 01-05.**  
El resto (testing, arquitectura, despliegue) puedes explorarlos en cualquier orden después de integración.

## Hitos del recorrido

### Hito 1: "Hola mundo" (Semana 2-3)
Tu primer programa funciona. Es simple, pero **es tuyo** y funciona.

### Hito 2: Primera interfaz (Semana 6-8)
Haces una página web que responde a clics. Ya no es estático, es interactivo.

### Hito 3: Primer servidor (Semana 10-12)
Creas un servidor que responde peticiones. Ya no solo haces frontends, haces lógica de servidor.

### Hito 4: Primera app con datos (Semana 14-16)
Guardas y recuperas datos de una base de datos. Ya no se pierde todo al refrescar.

### Hito 5: Primera app fullstack (Semana 18-20)
Frontend, backend y base de datos funcionando juntos. Ya no son piezas sueltas, es un sistema.

### Hito 6: Primera app online (Semana 22-24)
Tu aplicación está en internet con una URL. Otros pueden usarla. Ya no es solo local.

### Hito 7: Proyecto completo (Semana 26-28)
Tienes una aplicación completa, testeada, organizada y desplegada. Ya tienes portfolio.

## ¿Qué pasa si me pierdo?

**Estrategia de rescate:**

### Paso 1: Identifica dónde empezó la confusión
¿En qué ejercicio exacto dejó de tener sentido? Vuelve ahí.

### Paso 2: Repasa el módulo anterior
A veces el problema está en la base, no en el módulo actual.

### Paso 3: Simplifica
Vuelve al ejemplo más básico. Modifícalo poco a poco hasta entender cada parte.

### Paso 4: Pregunta (o explícalo en voz alta)
A veces el solo hecho de formular la pregunta te da la respuesta.

### Paso 5: Busca recursos adicionales
YouTube, documentación oficial, otros tutoriales. A veces necesitas escuchar la misma cosa de otra forma.

**No te rindas en el primer atasco.** El atasco es parte del proceso. La diferencia entre quien aprende y quien no, no es el talento. Es la persistencia.

## Personaliza tu recorrido

Este mapa es una guía, no una prisión.

### Si ya sabes algo:
Sáltalo o ve rápido. Usa el checklist del módulo para validar que realmente lo dominas.

### Si algo te interesa especialmente:
Profundiza en los recursos opcionales. Tómate tu tiempo.

### Si algo no te interesa ahora:
Haz lo esencial y sigue. Ya volverás si lo necesitas.

### Si tienes un proyecto en mente:
Úsalo como motivación. Pero sigue el orden de los módulos. Tu proyecto esperará y será mejor cuando tengas la base.
