# Checklist de Coherencia del Repositorio

Usa esta lista cada vez que añadas o revises un módulo.  
Marca con `[x]` lo que cumple.

---

## ✅ Estructura y presentación

- [ ] El módulo tiene una introducción que explica **qué se aprenderá** y **por qué es útil**
- [ ] Los conceptos nuevos se introducen **uno a la vez**, no en bloque
- [ ] Si hay contenido opcional, está **claramente marcado** como tal
- [ ] El módulo tiene ejercicios prácticos, no solo teoría

---

## ✅ Claridad y accesibilidad

- [ ] No se asumen conocimientos previos que no se hayan trabajado antes en el repo
- [ ] Las explicaciones usan **lenguaje simple** sin sacrificar precisión técnica
- [ ] Los ejemplos son **ejecutables** (el lector puede probarlos realmente)
- [ ] Hay **conexiones explícitas** a módulos anteriores cuando se usan conceptos previos

---

## ✅ Aprendizaje activo y error

- [ ] Los ejercicios invitan a **crear**, no solo a copiar
- [ ] Se incluyen **errores comunes** y cómo interpretarlos
- [ ] Hay sugerencias concretas para **experimentar y romper el código** sin miedo
- [ ] Se ofrecen pistas progresivas, no soluciones inmediatas

---

## ✅ Coherencia con el manifiesto

- [ ] El tono es **cercano y profesional**, sin paternalismo
- [ ] No se juzga el ritmo ni se etiquetan dificultades
- [ ] Se respeta el tiempo del lector (sin relleno innecesario)
- [ ] El módulo se puede **consultar después** sin necesidad de haberlo seguido linealmente

---

## ✅ Repetición y consolidación

- [ ] Si el concepto ya apareció antes, se **enlaza explícitamente** al módulo previo
- [ ] Si el concepto reaparece, se aporta un **ángulo nuevo o aplicación diferente**
- [ ] No se espera que el lector recuerde todo de memoria

---

## Cómo usar esta checklist

1. **Antes de publicar un módulo nuevo**, revisa cada punto
2. **Si algo no se cumple**, pregunta: ¿es realmente necesario? ¿Puedo ajustarlo?
3. **Si un punto no aplica** (ej: un módulo puramente teórico sin ejercicios), anótalo y justifica por qué
4. **Guarda esta checklist** junto al módulo como comentario o en una carpeta `/meta`

---

**Esta checklist no es rígida, pero mantiene visible el compromiso pedagógico del repositorio.**

Úsala incluso cuando no te apetezca. Especialmente entonces.
